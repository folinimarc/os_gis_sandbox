Info

The goal of OS Geostack Sandbox is to provide you with easy access to a variety of amazing open-source (OS) GIS software with minimal setup effort.

  • If not mentioned otherwise, data and configurations are persisted across Sandbox restarts.
  • While each component can be explored on its own, JupyterLab-GeoEnv contains tutorials and data stories (data driven cross-application tutorials) that provide guidance how different components might interact to transform raw data into meaningful real-world results.
  • It is possible to interact with Sandbox components from external software (such as QGIS) via the component's HostExternal and PortExternal connection information.
  • Some components such as JupyterLab-GeoEnv provide direct access to the host computer file system. This way processing scripts can even read and write data located outside of the Sandbox.

Each component of the Sandbox is a containerized application. Docker Compose manages the container lifecycle, the volumes used for data storage and the networking between the containers and towards the host. Content in form of tutorials and data stories is distributed as part of a data container, which copies its content to a volume accessible by other containers upon each start. The modular design makes it easy to add new components and new content.

This project is available on GitHub under MIT license here. Feel free to raise issues or fork the repository to contribute. There is also the possibility to provide direct feedback through a Microsoft Forms.
